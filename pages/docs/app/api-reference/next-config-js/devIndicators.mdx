# devIndicators

<AppOnly>

코드를 수정하고 Next.js가 애플리케이션을 컴파일할 때, 페이지의 오른쪽 하단에 컴파일 지시자가 나타납니다.

> **Good to know**: 이 지시자는 개발 모드에서만 나타나며, 프로덕션 모드에서 빌드하고 애플리케이션을 실행할 때는 나타나지 않습니다.

때때로 이 지시자가 페이지에 잘못된 위치에 나타날 수 있습니다. 예를 들어, 채팅 런처와 충돌할 때입니다. 위치를 변경하려면 `next.config.js` 파일을 열고 `devIndicators` 객체에서 `buildActivityPosition`을 `bottom-right`(기본값), `bottom-left`, `top-right` 또는 `top-left`로 설정하세요:

```js filename="next.config.js"
module.exports = {
  devIndicators: {
    buildActivityPosition: "bottom-right",
  },
};
```

때때로 이 지시자가 유용하지 않을 수 있습니다. 지시자를 제거하려면 `next.config.js` 파일을 열고 `devIndicators` 객체에서 `buildActivity` 설정을 비활성화하세요:

```js filename="next.config.js"
module.exports = {
  devIndicators: {
    buildActivity: false,
  },
};
```

또한 개발 중 빌드에서 페이지가 사전 렌더링될지 여부를 표시하는 지시자가 있습니다. 이는 페이지를 클릭하여 숨길 수 있지만, 항상 숨기고 싶다면 다음과 같이 설정을 비활성화할 수 있습니다:

```js filename="next.config.js"
module.exports = {
  devIndicators: {
    appIsrStatus: false,
  },
};
```

</AppOnly>

<PagesOnly>

> **Good to know**: 이 지시자는 Next.js 버전 10.0.1에서 제거되었습니다. 최신 버전의 Next.js로 업그레이드할 것을 권장합니다.

페이지가 [자동 정적 최적화](/docs/pages/building-your-application/rendering/automatic-static-optimization) 자격을 갖추면 이를 알려주는 지시자가 표시됩니다.

자동 정적 최적화는 매우 유익할 수 있으며, 개발 중에 페이지가 자격을 갖추는지 즉시 알 수 있는 것이 유용합니다.

때때로 이 지시자가 유용하지 않을 수 있습니다. 예를 들어, 전자 애플리케이션 작업 시 그렇습니다. 지시자를 제거하려면 `next.config.js` 파일을 열고 `devIndicators`에서 `autoPrerender` 설정을 비활성화하세요:

```js filename="next.config.js"
module.exports = {
  devIndicators: {
    autoPrerender: false,
  },
};
```

</PagesOnly>
